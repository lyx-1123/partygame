# 🚀 派對遊戲開發任務：AI 動態生成題庫版 (React)

請作為一名資深全端工程師，協助我使用 **React (Vite)**、**Tailwind CSS** 與 **Framer Motion** 開發一個派對猜謎遊戲網頁。

### 📋 1. 遊戲流程與畫面設計 (Screens)

#### A. 【設定畫面 (Setup Screen)】
- **功能：** 提供類別選擇（Dropdown）、目標答對題數 `targetGoal`（Input Number）、以及「開始遊戲」按鈕。
- **預載邏輯：** 點擊開始時，顯示 Loading 狀態，並同步呼叫 Gemini API 抓取首批 20 個詞彙。

#### B. 【遊戲畫面 (Game Screen)】
- **UI 佈局：**
    - 頂部：顯示目前進度（例如：第 5 / 10 題，僅計算答對的題數）。
    - 中央：大型詞彙卡片（Card），不顯示任何操作按鈕。
- **操作機制 (重點)：**
    - **電腦網頁：** 監聽鍵盤事件。按「右方向鍵 (→)」判定為 **Correct**；按「左方向鍵 (←)」判定為 **Pass**。
    - **手機裝置：** 監聽觸控手勢。**向右滑動 (Swipe Right)** 判定為 **Correct**；**向左滑動 (Swipe Left)** 判定為 **Pass**。
- **換題邏輯：**
    - 觸發操作後，紀錄該題結果與花費秒數。
    - 若判定為 Correct，則 `correctCount + 1`。
    - 若 `correctCount === targetGoal`，則自動結束遊戲進入結算畫面。

#### C. 【結算畫面 (Summary Screen)】
- **數據匯總：** 顯示總花費秒數。
- **清單展示：** 將出現過的題目分為「✅ 回答正確」與「❌ PASS」兩類顯示，並標註每一題的個別答題秒數。

---

### 🧠 2. AI 整合與題庫機制 (Gemini API)

- **API 服務：** 撰寫一個 `fetchAIWords(category)` 函式，呼叫 Gemini API。
- **System Prompt：** 
    > 你是一個遊戲題庫專家。請針對類別 [{category}] 產生 20 個適合派對遊戲的繁體中文名詞。
    > 難度需適中，避免過於冷門。
    > 請嚴格以 JSON 格式回傳，格式範例：`{"words": ["詞彙1", "詞彙2", ...]}`。不要包含任何解釋文字或 Markdown 語法。
- **無限補貨 (Buffering)：**
    - 內部維護一個 `wordList` 緩衝陣列。
    - 當 `currentIndex` 接近目前陣列末端（例如剩餘 5 個詞）時，自動在背景觸發 API 請求抓取新詞，並使用 `concat` 合併到陣列中，確保遊戲不中斷。
- **備用機制 (Fallback)：** 預設一組靜態常規題庫，若 API 呼叫失敗或 JSON 解析錯誤時自動遞補，確保遊戲流暢度。

---

### ⚙️ 3. 進階技術要求 (Performance & Interaction)

- **防抖動處理 (Debouncing/Throttling)：** 
    - 實作換題鎖定機制（Cooldown），在卡片滑出動畫結束前（約 300-400ms），忽略連續的按鍵或滑動輸入，避免狀態異常或重複計分。
- **動畫特效 (Framer Motion)：**
    - 實作卡片滑出效果：正確時向右飛出，PASS 時向左飛出。
    - 下一題卡片應以淡入或縮放效果進入畫面中央。
- **手勢偵測：**
    - 使用 `react-use-gesture` 或原生 `Touch` API 實現滑動判定。須設定合理的滑動距離門檻 (Threshold)，避免輕微晃動誤判。
- **計時器：**
    - 每一題顯示時重置 `startTime`。切換時計算 `(Date.now() - startTime) / 1000` 得到秒數。
